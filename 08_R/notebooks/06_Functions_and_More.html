Funciones en

Las funciones son objetos de R que evalan un conjunto de argumentos de entrada y devuelven una salida. Tambin son llamadas closures.

In[1]:
###################################
# Functions: creacin             #
###################################

func <- function(x) {
  #body  
}

Los argumentos son un conjunto de nombres de variables que se emplearn dentro de la funcin.

Los argumentos pueden ser:

  • Obligatorios.
  • Opcionales (tienen asignados un valor por defecto).
  • De longitud variable (se accede a cada uno de estos parmetros con ..1, ..2, etc.).

El retorno de la funcin se especifica llamando a la funcin return(x). En algunas ocasiones no es necesario, y se devolver el resultado de la ltima expresin evaluada. Es recomendable utilizarlo por legibilidad

In[9]:
do_something <- function(a, b = 1) {
    if (b == 0)
        return(0)
    a*b + a/b
}

do_something(4)
do_something(4,0)
8
0

La funcin anterior tiene un parmetro obligatorio (a) y otro opcional (b).

El parmetro b al ser opcional, si no es pasado en la llamada, tomar el valor 1.

La funcin tiene dos retornos:

  • Cuando b es igual a cero (con return)
  • Cuando b no es igual a cero (sin return)

Para realizar el retorno de la funcin hay que utilizar return(x). OJO! En R return es una funcin.

Es posible definir funciones en una nica lnea. En estos casos las llaves resultan opcionales.

In[10]:
pow_two <- function(x) return(x^2)

Dado que las funciones son objetos, se pueden pasar como argumentos a otras funciones (funciones de la familia apply).

En estos casos es muy comn utilizar funciones annimas.

Para ver los argumentos que recibe una funcin se puede utilizar la funcin args(f).

mbito de las variables en las funciones

Cuando una funcin es invocada se crea un nuevo entorno (frame) para ella.

Cada frame se encuentra dentro de la pila de llamadas, correspondiendo el primer frame al entorno global.

Cada llamada a una funcin crear un frame local.

Los nombres de las variables dentro de una funcin se resuelven en el siguiente orden:

  • Entorno local.
  • Entorno padre (funciones definidas dentro de funciones).
  • Entorno global.

De esta manera, una variable que est definida dentro de una funcin no est disponible fuera de la funcin (en los entornos superiores, aunque s en los inferiores).

Argumentos por valor

R pasa los argumentos por valor. Es decir, una funcin no puede alterar el valor de la variable que se pasa como argumento en la llamada.

Se generan copias locales de las variables en el mbito de la funcin. Es posible modificar el valor de una variable en los entornos superiores (global) desde un entorno local con el operador de asignacin <<-.

Los argumentos no tienen tipo, por lo que podemos pasar cualquier cosa en la llamada.

En R no es necesario pasar los argumentos en orden, puesto que los podemos pasar por nombre.

In[15]:
my_function <- function(a, b, c) {
  c(a,b,c)
    #d <<- c(a, b, c)
}

my_function(1, 2, 3)

my_function(c = 3, a = 1, b = 2)
  1. 1
  2. 2
  3. 3
  1. 1
  2. 2
  3. 3

Paquetes

Un paquete es una coleccin de funciones, datos y cdigo compilado que han sido empaquetados juntos.

Similares a las libreras de C/C++ o paquetes de clases de Java.

Para utilizar un paquete hay que instalarlo y a continuacin cargarlo:

  • install.packages(packageName)
  • library(packageName) o require(packageName)

En RStudio se pueden gestionar los paquetes desde la pestaa Packages

CRAN

Los paquetes de R se almacenan en un repositorio llamado CRAN (Comprehensive R Archive Network). Existen numerosas URLs para acceder al repositorio.

# Escribe una funcin en R, llamada cinco, que pida al usuario 5 
# nmeros por teclado. Si alguno de los nmeros introducidos es mayor o igual que
# 100, el programa devolver un mensaje avisando de esta situacin. Si no, devuelve
# la suma de los nmeros introducidos que sean menores que 10."""

cinco <- function(){
  
  num <- readline("Introduce 5 nmeros: ")
  num <-  as.numeric(as.vector(unlist(strsplit(num, ","))))
  if(any(num >100)){
    print('Hay al menos un nmero mayor que 100')
  }
  else{
    suma <- sum(num[num < 10])
    return(suma)
  }  
}

# Escribe una funcin que, dados un vector numrico x y un nmero natural M, realice el siguiente clculo: 
# Sumar una a una las componentes del vector
# x y detenerse una vez que pasemos de M. En este momento, devolver al usuario
# lo siguiente:
# a) La suma obtenida.
# b) Un vector que contenga los elementos de x que hemos usado para obtener
# dicha suma.


suma_vector <- function(x,M){
  suma = 0
  vector <- c()
  for(e in x){
    suma = suma + e
    vector <- c(vector,e)
    if(suma >= M){
      return(list(suma =suma,vector = vector))
      break
    }
  }
}
